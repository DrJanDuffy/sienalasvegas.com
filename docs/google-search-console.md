# Google Search Console — sienalasvegas.com

Setup and maintenance for [Google Search Console](https://search.google.com/search-console) so the site is verified, discoverable, and easy to monitor.

---

## 1. Add property

1. Go to [Google Search Console](https://search.google.com/search-console).
2. **Add property**: choose **URL prefix** and enter:
   - **`https://www.sienalasvegas.com`**
3. The domain must match `siteConfig.url` in `lib/site-config.ts` and your live deployment (e.g. Vercel).

**Optional:** Add both **www** and **non-www** as separate URL-prefix properties if you use redirects, so you can see coverage for each.

---

## 2. Verify ownership

### Option A: HTML meta tag (recommended)

1. In GSC, select **HTML tag** and copy the `content` value (e.g. `abc123xyz`).
2. In **Vercel** (or your host):
   - Project → **Settings** → **Environment Variables**
   - **Add:** Name `GOOGLE_SITE_VERIFICATION`, Value = the `content` value only (no `content="..."`).
   - Apply to **Production** (and Preview if you want).
3. **Redeploy** so the root layout emits the meta tag.
4. In GSC, click **Verify**. The app uses `metadata.verification.google` in `app/layout.tsx` when `GOOGLE_SITE_VERIFICATION` is set.

### Option B: HTML file upload

1. In GSC, pick **HTML file** and download the file (e.g. `google123abc.html`).
2. Place it in the project **`public/`** folder.
3. Deploy; the file will be at `https://www.sienalasvegas.com/google123abc.html`.
4. Click **Verify** in GSC.

### Option C: DNS TXT record

If you use Cloudflare (or another DNS host), add the TXT record GSC provides. No code or deploy needed. Use **DNS only** (gray cloud) with Vercel to avoid SSL conflicts.

## 3. Sitemap

- **URL:** `https://www.sienalasvegas.com/sitemap.xml`
- **Source:** Generated by `app/sitemap.ts` at build time (Next.js `MetadataRoute.Sitemap`).
- **Discovery:** The root layout includes `<link rel="sitemap" type="application/xml" href=".../sitemap.xml" />` so crawlers can find it.

**Submit in GSC:**

1. Left menu → **Sitemaps**.
2. Under **Add a new sitemap**, enter: **`sitemap.xml`**
3. Submit. GSC will show status and any errors.

**Included:** Core pages, services, buyer/seller personas, 55+ communities (including Siena subpages), neighborhoods, FAQ, market report/update/insights, google-business, security-policy. `lastModified` is set at build; priorities and `changeFrequency` are set per section.

---

## 4. robots.txt

- **URL:** `https://www.sienalasvegas.com/robots.txt`
- **Served file:** Next.js serves **`public/robots.txt`** if present (it takes precedence over `app/robots.ts`). The repo has a static `public/robots.txt` with:
  - `User-agent: *` → `Allow: /`, `Disallow: /api/`, `/admin/`, `/_next/`, `/monitoring/`
  - `Sitemap: https://www.sienalasvegas.com/sitemap.xml`
  - Optional rules for AI crawlers and crawl-delay for some bots.
- **Programmatic fallback:** `app/robots.ts` generates allow `/`, disallow `/api/`, and the same sitemap URL; use it if you remove `public/robots.txt`.

Ensure the sitemap URL in `public/robots.txt` matches `siteConfig.url` (e.g. `https://www.sienalasvegas.com`).

## 5. After setup — ongoing use

- **URL Inspection:** Use **Inspect any URL** to check indexing status and **Request indexing** for new or updated key pages (home, listings, new neighborhood/55+ pages).
- **Coverage / Pages:** Fix any “Excluded” or “Error” URLs (e.g. redirects, soft 404s, canonical issues). Canonicals are set per page via `buildPageMetadata()`.
- **Performance:** Review queries, clicks, and impressions; align titles/descriptions with high-potential queries.
- **Core Web Vitals:** Use the **Experience** report; fix any “Poor” URLs (mobile/desktop). The site uses lazy-loaded scripts and image optimization to support LCP/CLS.
- **Enhancements:** Check **Structured data** for errors. The site uses LocalBusiness/RealEstateAgent, WebSite, and page-specific schema (FAQ, HowTo, WebPage, etc.) from `lib/schema.ts` and `SchemaScript`.
