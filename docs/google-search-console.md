# Google Search Console — sienalasvegas.com

## 1. Add property

1. Go to [Google Search Console](https://search.google.com/search-console).
2. **Add property**: choose **URL prefix** and enter `https://www.sienalasvegas.com` (must match your live domain and `metadataBase` in the app).

## 2. Verify ownership

### Option A: HTML meta tag (recommended)

1. In GSC, pick **HTML tag** and copy the `content` value (e.g. `abc123...`).
2. In Vercel (or your host), add an **environment variable**:
   - **Name:** `GOOGLE_SITE_VERIFICATION`
   - **Value:** the `content` value only (no `content="..."`).
3. Redeploy so the root layout emits the meta tag, then click **Verify** in GSC.

### Option B: HTML file upload

1. In GSC, pick **HTML file** and download the file (e.g. `google123abc.html`).
2. Put it in the project’s **`public/`** folder.
3. Deploy; the file will be at `https://www.sienalasvegas.com/google123abc.html`.
4. Click **Verify** in GSC.

### Option C: DNS TXT record

If you use a DNS host (e.g. Cloudflare), add the TXT record GSC shows. No code changes.

## 3. Submit sitemap

After verification:

1. In GSC, open **Sitemaps** (left menu).
2. Under **Add a new sitemap**, enter: `sitemap.xml`
3. Submit. The sitemap URL is: **`https://www.sienalasvegas.com/sitemap.xml`**

The app generates this from `app/sitemap.ts` and includes all main pages plus Siena subpages and 55+ community pages.

## 4. robots.txt

- **URL:** `https://www.sienalasvegas.com/robots.txt`
- Generated by `app/robots.ts` (allows `/`, disallows `/api/`, points to the sitemap).
- A static `public/robots.txt` is also present as a fallback.

## 5. After setup

- Use **URL Inspection** to request indexing for important URLs.
- Use **Performance** to see queries and pages; fix coverage issues if any.
- Keep NAP and LocalBusiness schema consistent with your Google Business Profile.
